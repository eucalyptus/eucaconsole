Scaling Group Detail - Instances

This dialog allows you to create a scaling group. An Auto Scaling group defines the parameters for the Eucalyptus instances that are used for scaling, as well as the minimum, maximum, and (optionally) the desired number of instances to use for Auto Scaling your application. For more information on Auto Scaling, please see Using Auto Scaling in the Eucalyptus User Guide.

Instances

This section allows you to view and manage instaces in the auto scaling group.

Context menu actions

Each entry in the policies list has a context menu, accessible by clicking on the gear icon, with actions that you can perform on the selected policy.

The following context menu actions are availabe:

View details

This item allows you to view the details page for the selected instance.

Mark unhealthy

This item allows you to mark an instance as unhealthy. This will cause auto scaling to terminate the instance and launch a new instance to replace it.

Mark unhealthy

This action terminates the selected instance.
